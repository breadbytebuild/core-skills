# Meta-Cognitive & Thinking Skills

The foundation layer. These skills don't solve specific problems — they change how you approach *all* problems. An agent with strong meta-cognitive skills will make better decisions across every domain.

## Skills

- **Design Thinking** — Human-centered problem solving through empathy, ideation, and iteration
- **First Principles** — Break problems down to fundamental truths and reason up from there
- **Systems Thinking** — Understand complex systems through feedback loops, emergent behavior, and leverage points
- **Structured Brainstorming** — Force genuine idea diversity through structured lenses and constraints
- **Critical Evaluation** — Stress-test ideas against evidence, counteracting sycophancy and confirmation bias
- **Prioritization Frameworks** — Decide what matters most and kill everything else, counteracting "everything is important" paralysis

## How They Work Together

These skills are designed to compose. Each one sharpens a different phase of thinking. Here's the flow for a complex problem:

```
First Principles    →  What's actually true? Strip assumptions.
Design Thinking     →  What's the right problem? Frame it for real humans.
Systems Thinking    →  How does it all connect? Map feedback loops.
Brainstorming       →  What are the options? Generate diverse solutions.
Critical Evaluation →  Are these ideas sound? Stress-test each one.
Prioritization      →  What do we actually do? Rank and kill the rest.
```

You don't always need all six. Pick the ones that match the task. But for high-stakes, ambiguous problems, the full stack produces dramatically better outcomes than any single skill alone.

## Worked Example: The Skills in Action

**Scenario:** A startup's user retention is dropping. The CEO says "we need to add more features."

**First Principles:** Is "more features" actually the right response? Strip the assumption. Bedrock truth: users leave when the product doesn't deliver enough value relative to alternatives. "More features" is convention, not bedrock. The real question is: why are users not getting value?

**Design Thinking (Discover → Define):** Walk the user journey. A new user signs up, sees the dashboard, tries one feature, doesn't get a result in the first session, and never comes back. The Empathy Protocol reveals: they're not frustrated — they're *confused*. They don't know what to do first. The reframe: the problem isn't missing features. It's that users never reach the value the existing features already provide.

**Systems Thinking:** Map the loop. More features → more complex UI → more confusion for new users → lower retention → pressure to add more features to "fix" retention. It's a reinforcing loop making the problem worse. The leverage point isn't features — it's the onboarding experience that determines whether users ever reach value.

**Structured Brainstorming (Develop):** Use the Lens Toolkit. Constraint Injection: "What if we could only show 1 feature on first login?" Inversion: "How would we make onboarding *worse*?" (more options, more text, more steps). Reverse each answer. Perspective Shift: "How would a game designer onboard a new player?" (progressive disclosure, small wins, clear next step). This produces 5+ genuinely distinct approaches to onboarding — not 5 variations of "add a tutorial."

**Critical Evaluation (Deliver):** Pre-mortem the top 2 approaches. Run the Evidence Audit — what's verified (users drop off in session 1) vs. assumed (they'd stay if onboarding were better)? Steel man the CEO's original "more features" argument — is there any scenario where that's right? (Maybe: if power users are the ones churning, features might matter. Check the data.)

**Prioritization:** ICE score the surviving options. The guided onboarding redesign scores highest (high impact, high confidence from data, moderate ease). The "reduce to 1 feature on first login" is high impact but low confidence. Kill the feature additions — not because they're bad ideas, but because onboarding is more important right now. Explicitly state: "We're not building new features this quarter because fixing onboarding will have 3x the retention impact."

**Result:** Instead of building more features (which would have worsened the loop), the team ships a focused onboarding redesign. One decision, grounded in six layers of thinking, that addresses the actual problem instead of the obvious symptom.
