# Meta-Cognitive Skills — Routing Index

When you encounter a situation that matches a trigger below, load the corresponding skill file and follow its process. You don't need to load all skills — pick the ones that match the task. For complex, high-stakes problems, load multiple skills and follow the composition flow at the bottom.

## Skill Triggers

**Design Thinking** → `skills/meta-cognitive/design-thinking.md`
Load when: the problem is ambiguous, real humans are affected, you're building something new, the obvious solution feels too easy, or a stakeholder says "build X" but you sense the real need is different.

**First Principles** → `skills/meta-cognitive/first-principles.md`
Load when: someone says "that's just how it's done," a constraint seems immovable but you can't explain why from fundamentals, you're copying an approach without understanding the reasoning, or the problem feels stuck.

**Systems Thinking** → `skills/meta-cognitive/systems-thinking.md`
Load when: the same problem keeps recurring despite being "fixed," a fix in one area creates problems elsewhere, there are conflicting incentives among stakeholders, or past interventions produced unintended consequences.

**Structured Brainstorming** → `skills/meta-cognitive/structured-brainstorming.md`
Load when: you need creative options and your first ideas all look similar, the obvious approach isn't working, or someone asks for "alternatives" and you're about to list minor variations.

**Critical Evaluation** → `skills/meta-cognitive/critical-evaluation.md`
Load when: evaluating a plan before committing, choosing between options, someone presents a confident conclusion, you notice yourself agreeing without independent reasoning, or the stakes are high enough that being wrong matters.

**Prioritization Frameworks** → `skills/meta-cognitive/prioritization-frameworks.md`
Load when: you have more options than you can pursue, everything feels equally important, you're about to say yes without considering what it displaces, or you need to decide what to do first (and what to kill).

## Composition Flow

For complex problems, use this sequence:

```
1. First Principles    →  Strip assumptions. What's actually true?
2. Design Thinking     →  Frame the problem. Who is affected and why?
3. Systems Thinking    →  Map connections. What loops? What are second-order effects?
4. Brainstorming       →  Generate options. Force diversity, not just volume.
5. Critical Evaluation →  Test options. Is each one sound? (soundness)
6. Prioritization      →  Rank and cut. Which sound options matter most? (importance)
```

You don't always need all six. But for high-stakes, ambiguous problems, the full stack produces dramatically better outcomes than any single skill alone. Load the ones that match the phase you're in.

## Quick Decision Guide

- **"What should we build?"** → Design Thinking + First Principles
- **"Why isn't this working?"** → Systems Thinking + First Principles
- **"Give me some ideas"** → Structured Brainstorming
- **"Is this plan good?"** → Critical Evaluation
- **"What do we do first?"** → Prioritization Frameworks
- **"This problem is complex and high-stakes"** → Load all six, follow the composition flow
