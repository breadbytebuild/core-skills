# App Brainstorming — Market Analysis & Feature Remix Engine

## How This Works

Every successful app is built from a small set of *feature primitives* — psychological mechanics that drive engagement, retention, and habit formation. Streaks aren't a Duolingo invention. They're a *loss aversion primitive* that works in language learning, fitness, sobriety, prayer, meditation, and anywhere habits matter. The companion isn't a Finch invention. It's an *emotional attachment primitive* that works in self-care, prayer, productivity, and parenting.

The skill is to see through the surface (a green owl, a virtual bird) to the underlying mechanic (loss aversion, nurture instinct), then apply that mechanic to a new vertical with a new skin.

This skill teaches three capabilities:
1. *Tear apart a market* — identify winners, map their mechanics, find gaps
2. *Speak the language of primitives* — understand the ~65 proven feature mechanics and WHY they work psychologically
3. *Remix and recombine* — generate novel app concepts by applying proven mechanics to new verticals

## Phase 1: Market Teardown

Before brainstorming what to build, understand what exists. Load `references/market-teardown.md` for the full framework.

*Quick version:*
1. *Map the landscape* — list the top 10-20 apps in the category by downloads, revenue, and ratings
2. *Audit their mechanics* — for each winner, identify which feature primitives they use (reference the primitive library)
3. *Map the psychology* — what core human needs does this category serve? (belonging, progress, safety, identity, etc.)
4. *Find the gaps* — which proven primitives are NO ONE using in this category? That's your opportunity
5. *Analyze demographics* — who are the users? What OTHER apps do they love? Those cross-category insights are gold

## Phase 2: Feature Primitive Library

Load `references/feature-primitives.md` for the full database of ~65 proven mechanics organized by psychological mechanism.

*The categories:*
- *Habit Formation* — streaks, daily rituals, routine anchoring
- *Emotional Attachment* — companions, growth visualization, customization
- *Loss Aversion* — streak protection, decay, limited attempts
- *Progress & Achievement* — XP, levels, badges, progress bars, skill trees, unlocks
- *Competition & Social* — leaderboards, challenges, accountability, activity feeds
- *Reward Systems* — variable rewards, daily bonuses, virtual currency, loot, celebrations
- *Crisis & Support* — panic buttons, SOS modes, guided calming, live help
- *Content & Learning* — micro-lessons, daily drips, spaced repetition, quizzes
- *Identity & Commitment* — public commitment, identity labels, day counters, milestone sharing
- *AI & Personalization* — AI coach, adaptive difficulty, smart recommendations, contextual triggers
- *Community* — forums, peer matching, mentorship, user stories
- *Journaling & Reflection* — guided journaling, mood tracking, gratitude, time capsules, year reviews
- *Notification Design* — mascot reminders, streak alerts, social triggers, smart timing
- *Visualization & Data* — heat maps, stat dashboards, before/after, wrapped reviews
- *Gamification Structures* — quests, daily challenges, timed events, collection systems

Each primitive includes: what it is, the psychology behind it, best-in-class examples, and remix potential.

## Phase 3: The Remix Engine

This is the core brainstorming methodology. Load `references/remix-playbook.md` for the full framework with case studies.

### The 5-Step Remix

1. *Pick a vertical.* Define the niche, demographic, and core human need it serves.
   - Example: "Prayer habits for Christian women 25-45"
   - The need: spiritual connection, consistency, community, peace

2. *Find the analogue.* Which app in a DIFFERENT category serves a similar psychological need?
   - Prayer consistency ≈ meditation consistency ≈ language learning consistency
   - Spiritual companion need ≈ self-care companion need (Finch)
   - Prayer community ≈ fitness community (Strava)

3. *Extract the primitive.* Strip the analogue to its mechanic. Remove the domain-specific skin.
   - Finch → "Nurture a creature that grows with your consistency" → *Companion + Growth Visualization*
   - Duolingo → "Track consecutive days to build loss aversion" → *Streak + Loss Aversion*
   - Strava → "See your friends' activity and feel motivated" → *Social Activity Feed*

4. *Re-skin for the vertical.* Apply the naked primitive to the new domain.
   - Companion for prayer → "Little Lamb" that grows as you pray daily (FaithTime did this)
   - Streak for sobriety → Day counter that's emotionally painful to reset
   - Activity feed for recovery → See your support group's check-ins anonymously

5. *Combine 3-5 primitives into a concept.* A single primitive isn't an app. A coherent combination of 3-5 primitives serving the same core need IS an app.

### The Remix Validation Checklist

Before declaring a concept viable:
- [ ] Does the combination serve ONE clear core need? (scattered primitives = scattered app)
- [ ] Does each primitive earn its place? (can you explain WHY this mechanic for THIS audience?)
- [ ] Is there a hook loop? (Trigger → Action → Variable Reward → Investment)
- [ ] Would this demographic actually want this? (not every mechanic works for every audience — gamification can feel patronizing to some)
- [ ] What's the "Day 1 magic moment"? (the thing that happens in the first session that makes someone come back)
- [ ] Can you articulate the concept in one sentence? ("It's [analogue] but for [vertical]")
- [ ] Is anyone else already doing this exact combination? (if yes, what's your wedge?)

## Phase 4: Concept Output Format

Every brainstormed concept should be structured as:

```
## [Concept Name]

**One-liner:** [Analogue] meets [Analogue] for [Demographic]
**Core need:** [The fundamental human need this serves]
**Target:** [Demographic with specifics]

**Primitive Stack:**
1. [Primitive] — [How it manifests in this app] (borrowed from [Source App])
2. [Primitive] — [How it manifests] (borrowed from [Source])
3. [Primitive] — [How it manifests] (borrowed from [Source])

**Day 1 Magic Moment:** [What happens in the first session]
**Hook Loop:** Trigger → Action → Reward → Investment
**Monetization:** [How this makes money]
**Competitive Wedge:** [Why this wins vs. existing options]
**Risk:** [The biggest reason this might not work]
```

## Generating Multiple Concepts

When brainstorming, generate *at least 5 concepts* per vertical using different primitive combinations. Use the Structured Brainstorming skill's diversity techniques:
- *Constraint injection:* "What if it had zero social features?" "What if it was audio-only?"
- *Perspective shift:* "What would a game designer build?" "What would a therapist build?"
- *Inversion:* "What would make people HATE a [category] app?" (then reverse each answer)
- *SCAMPER on the leader:* Take the #1 app, run Substitute/Combine/Adapt/Modify/Put to other use/Eliminate/Reverse

After generating 5+ concepts, rank them by:
1. *Market gap size* — how underserved is this combination?
2. *Primitive-audience fit* — how natural do these mechanics feel for this audience?
3. *Defensibility* — can you build moats (community, data, content, brand)?
4. *Business model clarity* — is monetization obvious?
5. *Build complexity* — can a small team ship v1 in 3 months?

## Anti-Patterns

| Pattern | Fix |
|---------|-----|
| "It's Uber for X" with no primitive analysis | Name the specific mechanics. "Uber for X" isn't a concept, it's a shorthand that hides sloppy thinking. |
| Stacking 10+ primitives into one app | 3-5 max. More is feature bloat, not innovation. Each one must earn its place. |
| Picking primitives you like instead of ones the audience needs | Start from the audience's psychology, not from cool mechanics. |
| Ignoring that someone already did this | Always check. The Bible/Finch remix worked because nobody had done it yet in that vertical. |
| Surface-level remix ("add streaks to everything") | Go deeper than streaks. The primitives library has 65 options. If your remix is just "X + streaks," you're not thinking hard enough. |
| Skipping the market teardown | You can't find gaps if you don't know the landscape. Phase 1 before Phase 3, always. |
