# Product Building Skills — Routing Index

Skills for building great software, organized by where you are in the product lifecycle. Load the skill that matches your current phase.

## Phase 1: Define (before you build)

**Product Specs** → `skills/product-building/product-specs.md`
Load when: writing a PRD, defining acceptance criteria, scoping a feature, or creating a task spec that another person or agent needs to execute from. *Coming soon.*

**Stakeholder Communication** → `skills/product-building/stakeholder-comms.md`
Load when: updating PMs, designers, or founders on progress, presenting tradeoffs, or requesting a decision on scope or direction. *Coming soon.*

## Phase 2: Design (how it should look and feel)

**UI/UX Design Sense** → `skills/product-building/ui-ux-design.md`
Load when: designing a screen, choosing layouts, making spacing/typography/color decisions, or anything where the visual quality of the software matters. Load this before building any UI.

**Frontend Craft** → `skills/product-building/frontend-craft.md`
Load when: building components, handling loading/error/empty states, implementing animations or transitions, or making responsive design decisions. *Coming soon.*

## Phase 3: Build (writing the code)

**Code Quality** → `skills/product-building/code-quality.md`
Load when: writing new code, refactoring, naming things, organizing files, or any time you want the code to be maintainable, not just functional. *Coming soon.*

**Debugging** → `skills/product-building/debugging.md`
Load when: something is broken, a test is failing, behavior is unexpected, or you've tried a fix and it didn't work. Especially when you catch yourself guessing instead of tracing. *Coming soon.*

## Phase 4: Ship (getting it out the door)

**PR Writing** → `skills/product-building/pr-writing.md`
Load when: creating a pull request, writing a commit description, or summarizing changes for review. *Coming soon.*

**Software Polish** → `skills/product-building/software-polish.md`
Load when: a feature is "done" but doesn't feel premium yet. The final pass for micro-interactions, copy, edge cases, performance, and accessibility. *Coming soon.*

## Phase 5: Maintain (after it's live)

**Scaling & Architecture** → `skills/product-building/scaling-architecture.md`
Load when: making architecture decisions, optimizing performance, deciding whether to refactor or patch, or planning for growth. *Coming soon.*

**Copywriting** → `skills/product-building/copywriting.md`
Load when: writing user-facing text (error messages, button labels, onboarding copy, empty states, tooltips). Any words a user reads. *Coming soon.*

## Quick Decision Guide

- **"Write a spec for this feature"** → Product Specs
- **"Design this screen"** → UI/UX Design Sense
- **"Build this component"** → Frontend Craft + Code Quality
- **"This is broken"** → Debugging
- **"Create a PR"** → PR Writing
- **"It works but looks rough"** → Software Polish
- **"Should we refactor this?"** → Scaling & Architecture
- **"Write the error message"** → Copywriting
